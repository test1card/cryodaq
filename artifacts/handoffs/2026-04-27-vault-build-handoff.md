## What didn't

- **Codex CLI sandbox-write rejection** — known per skill §6, but
  required the stdout-redirect workaround on every Codex dispatch.
  Long stretches of "Could not update ... read-only sandbox" prose
  in the response file before the canonical findings; tail parsing
  is the operating mode.
- **Hook false-positives.** This environment's hooks intermittently
  emit "Edit operation failed" / "Write operation failed" while the
  tool itself reports success and grep confirms changes landed.
  Treated as known false-positive; verified actual state with
  ls / wc / grep after each batch.
- **Gemini contradiction across rounds** (Т4 alarm-group inclusion).
  Lesson: even with consistent wide-context, Gemini's structural
  reading can flip between rounds. Repo state is the tiebreaker.
