## Audit history

**Codex (gpt-5.5, high reasoning):** FAIL → PASS (2 cycles)

Cycle 1 findings and fixes:
- CRITICAL: Engine never called `add_sample()` — measurement window always returned "Insufficient samples" in production. → Fixed: `_leak_rate_feed()` async task subscribes to DataBroker, filters `unit=="mbar"`, calls `add_sample()` when estimator active.
- CRITICAL: `add_sample()` accumulated all samples without trimming. → Fixed: sliding window trims samples with `t_rel < (latest_t_rel - window_s)`.
- HIGH: `ss_xx == 0` in OLS returned `r2=1.0` (undefined regression reported as perfect fit). → Fixed: returns `r2=0.0`.
- HIGH: `finalize()` didn't clear `_samples` — second call returned stale data. → Fixed: `samples = self._samples; self._samples = []` before fitting.
- MEDIUM: `duration_s` unvalidated — string, NaN, inf, zero accepted. → Fixed: float conversion + `not (0 < window_s < inf)` guard.
- LOW: log line used `len(self._samples)` after clear → logged `n=0`. → Fixed: uses `len(samples)`.
- Additionally: auto-finalize not wired. → Fixed in same cycle: `_leak_rate_feed()` calls `should_finalize()` and runs `finalize()` + `event_logger.log_event()` on expiry.
- Additionally: no pressure channel filter → mixing multi-gauge streams. → Fixed: uses `_vt_cfg.get("pressure_channel", "")`.

Cycle 2: PASS, no new findings.
