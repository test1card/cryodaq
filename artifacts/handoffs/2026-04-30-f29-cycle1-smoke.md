## Harness Fix During Smoke

First smoke attempt exposed a harness issue: fake timer sleep returned forever,
so `_periodic_report_tick` looped rapidly and filled the GUI subscriber queue.
The smoke harness now raises `CancelledError` on the second sleep, publishing
exactly one scheduled request per active/idle check.

Second smoke attempt exposed a real prompt issue: the periodic system prompt
hardcoded "последний час", causing a 15-minute configured window to be
summarized as "за последний час". Fixed in
`PERIODIC_REPORT_SYSTEM` by changing the task wording to "заданное окно
времени"; regression covered by
`test_periodic_report_prompt_does_not_hardcode_hour_window`.

The smoke harness is not a full engine `--mock` process. It exercises the

inference -> Telegram/log/GUI/audit dispatch -> idle skip.
