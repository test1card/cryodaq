## Cross-cutting observations

### Stub label leak (Scenarios 2 & 4)

Root cause: stub text `[Активные алармы — wired in Cycle 4 — historical SQLite context]` 
appears in the prompt context. gemma4:e4b picks it up and references it in output.

Fix path: wire real alarm history in Cycle 4. The stubs are placeholders with honest
framing — the model is working correctly. Not a prompt engineering issue.

Workaround for demo before Cycle 4: replace stubs with neutral Russian:
`"нет данных"` or `"алармов не зафиксировано"` — model won't echo neutral placeholders.

### Shift handover latency (92.6s)

The shift handover generates the longest response (933 chars / 796 tokens) — the model
reasons extensively and produces a structured 3-section summary. Thinking chain is large.

Options:
- Accept as-is (within 120s timeout, async so operator not blocked)
- Try `num_ctx: 4096` — likely cuts prefill from ~22s to ~5s, giving headroom
- Note: Cycle 4 context will add real alarm history → even more tokens → watch latency
