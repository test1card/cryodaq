# ADR 001: Agent-native plugin contract and conformance are one interface

- Status: Accepted for the post-Montana roadmap phase
- Date: 2026-07-21
- Implementation status: Deferred; this ADR adds no plugin product code

## Context

CryoDAQ has repeatedly preserved rules that were asserted by executable tests
and drifted where architecture or wiring existed only in prose. A weaker model
cannot safely reconstruct the repository's implicit extension architecture on
every task. The inference burden must move into a versioned contract, a
conformance floor, and a conformant starting scaffold.

## Decision

`PLUGIN_CONTRACT.md`, the plugin schema, `tests/conformance/`, the public plugin
registry, and `plugins/_template/` will be one versioned interface. None may
change alone.

Every obligating contract clause has a stable ID. A machine-readable registry
maps obligation IDs to named conformance checks. A meta-test enforces exact set
equality and rejects duplicate IDs, missing checks, orphan checks, skips, and
expected failures used in place of enforcement. A prose-only rule is guidance
and an open conformance gap; it is not an added obligation.

The suite is plugin-independent and discovers plugins only through the public
registration surface. It covers exact Reading type/shape/unit/identity,
hostile-input rejection, golden physical vectors, safety-manifest schema,
bounded lifecycle, and panel design-system behavior. Its seeded failure corpus
must catch weak-model errors: wrong types/shapes, swallowed hostile input,
silent drops, physically wrong scale/endianness/decimation despite valid
syntax, and over-claimed authority.

Panel conformance is executable. It checks canonical tokens/components,
Russian operator copy, non-color cues, keyboard behavior, and the independent
availability/freshness/connectivity and safety-severity axes. Referencing an
aspirational design-system tool that does not exist grants no evidence.

`tests/docs/test_docs_freshness.py` will enforce an exact structured inventory
between tracked `src/cryodaq/**/*.py` modules and architecture entries, plus
contract/template/registration version consistency. Fuzzy prose search is not
the architecture registry.

The phase starts only after Montana convergence. A half-migrated repository
teaches inconsistent idioms and is not a valid agent-generation substrate.

## Acceptance

The template and one real passive instrument plugin generated by a
representative mid-tier model must reach the same final conformance result as a
stronger reference model. Only first-try success and repair count may differ.
Seeded bad plugins must fail for their intended obligation IDs.

## Consequences

Contract changes cost more because they require coordinated schema, harness,
template, registry, documentation, and version updates. In return, plugin
quality no longer depends on trusting a model's architectural inference or
self-review. The future phase must implement all named checks before claiming
this decision is enforced.
