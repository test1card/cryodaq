# Icon Tokens

CryoDAQ uses **Lucide icon set** (MIT licensed), bundled as SVG files. Icons are monochrome, inherit current text color, and scale cleanly at any size.

Icon sizing tokens are PROPOSED — not yet in `theme.py`. Current widget code hardcodes sizes. This document specifies the target token set.

## Icon source

**Lucide** (https://lucide.dev/) — chosen because:
- MIT licensed (permissive, production-safe)
- Consistent stroke-based style (matches industrial aesthetic — no decorative flourishes)
- ~1200 icons, covers all CryoDAQ needs (nav, actions, status, equipment)
- SVG-first (crisp at any scale, small file size)
- 2px stroke width standard (pairs with our 4px spacing base)

Icons bundled at `src/cryodaq/gui/assets/icons/` as individual SVG files.

**Do NOT use:**
- Font Awesome (mixed licensing, heavier)
- Material Icons (wrong visual language — too playful/filled)
- Emoji as icon — **strict prohibition** (see below)

## Sizing scale (proposed)

| Token (proposed) | Value (px) | Use |
|---|---|---|
| `ICON_SIZE_SM` | `16` | Inline with BODY text (14px) — icon slightly larger than text cap-height |
| `ICON_SIZE_MD` | `20` | Default icon size — buttons, tool rail, labels |
| `ICON_SIZE_LG` | `24` | Emphasized icons — large buttons, category headings |
| `ICON_SIZE_XL` | `32` | Display icons — empty state illustration replacement, large buttons |

**No `ICON_SIZE_XS`.** Icons below 16px lose legibility at lab PC viewing distance. If you need smaller visual element, use a circular status dot (4-8px) instead.

## Stroke width

| Token (proposed) | Value (px) | Use |
|---|---|---|
| `ICON_STROKE_WIDTH` | `2` | Default stroke for all Lucide icons |

Matches Lucide's default export width. Do not override — changing stroke width creates inconsistent icon weight across the UI.

## Color inheritance

Icons MUST inherit their color from surrounding text, not carry their own hue.

## Status + icon pairing rule

**Status information is NEVER conveyed by color alone.** Every colored status indicator must include an icon (or shape) that redundantly communicates the status.

| Status | Color | Required icon |
|---|---|---|
| OK | `STATUS_OK` green | `check-circle` |
| Warning | `STATUS_WARNING` amber | `alert-triangle` |
| Caution | `STATUS_CAUTION` orange | `alert-triangle` (same as warning, filled) |
| Fault | `STATUS_FAULT` red | `x-octagon` |
| Stale | `STATUS_STALE` gray | `cloud-off` or `wifi-off` |
| Info | `STATUS_INFO` blue | `info` |
| Cold | `COLD_HIGHLIGHT` blue | `snowflake` |

See `rules/accessibility-rules.md` RULE-A11Y-002 (color not sole channel).

## Emoji prohibition

**Emoji are forbidden as functional icons in operator UI.** This includes 🔔, ⚠️, ✅, ❌, and all emoji.

Rationale:
- **Cultural inconsistency** — emoji rendering varies across OS (Apple vs Windows vs Linux). 🔔 looks different on operator's lab PC vs dev's MacBook.
- **Lack of semantic control** — emoji carry connotations (🔔 = "cute notification", wrong for engineering alarm)
- **Wrong aesthetic** — emoji are playful; CryoDAQ is instrumentation
- **Encoding risk** — some terminals / log readers may strip emoji
- **Accessibility** — screen readers announce emoji unpredictably

Use Lucide SVG icons for all functional icon needs. A bell icon for "notifications" is `bell.svg` (Lucide), not 🔔.

**Exception:** emoji MAY appear in user-typed content (operator log notes, experiment descriptions) — user input is preserved verbatim. Emoji never appear in UI chrome, labels, or status indicators generated by CryoDAQ code.

See `rules/content-voice-rules.md` RULE-COPY-005.

## Icon + text alignment

When icon appears inline with text:

```python
# DESIGN: RULE-SPACE-008 (icon vertical alignment with text)
row = QHBoxLayout()
row.setSpacing(theme.SPACE_1)  # 4px between icon and text
row.setAlignment(Qt.AlignmentFlag.AlignVCenter)

icon = QLabel()
icon.setFixedSize(theme.ICON_SIZE_SM, theme.ICON_SIZE_SM)  # proposed: theme.ICON_SIZE_SM — not yet in theme.py
icon.setPixmap(load_colored_icon("alert-triangle").pixmap(theme.ICON_SIZE_SM, theme.ICON_SIZE_SM))  # proposed: theme.ICON_SIZE_SM — not yet in theme.py

label = QLabel("Внимание: температура выше 77K")
label.setFont(body_font)

row.addWidget(icon)
row.addWidget(label)
```

Icon cap-height should roughly match text cap-height. For 14px body text, 16px icon (ICON_SIZE_SM) works. For 20px XL heading, 24px icon (ICON_SIZE_LG) works.

## Icon-only buttons

Buttons with only icon (no text) MUST have tooltip (see RULE-INTER-008). Minimum button size for icon-only is 32×32px (`ROW_HEIGHT - 4` for toolrail-style compact buttons is 32, default ROW_HEIGHT 36 accepts centered 20px icon).

```python
# DESIGN: RULE-INTER-008 (icon-only button requires tooltip)
close_button = QPushButton()
close_button.setFixedSize(32, 32)
close_button.setIcon(load_colored_icon("x", color=theme.MUTED_FOREGROUND))
close_button.setIconSize(QSize(theme.ICON_SIZE_MD, theme.ICON_SIZE_MD))  # proposed: theme.ICON_SIZE_MD — not yet in theme.py
close_button.setToolTip("Закрыть (Esc)")  # MANDATORY
```

## Anti-patterns

- **Using emoji as icon** — strict prohibition
- **Font Awesome or Material Icons mixed in** — inconsistent icon language
- **Icon with hardcoded fill color** — breaks theme adaptation
- **Icon without tooltip on icon-only button** — unlearnable UI
- **Icon sizes not from scale** — `setIconSize(18)` when scale has 16 and 20
- **Filled/bold icon weight** — use Lucide default stroke style only

See `ANTI_PATTERNS.md#icons`.
