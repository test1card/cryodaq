# Руководство оператора CryoDAQ

Версия системы: 0.1.0
Дата: март 2026

---

## 1. Назначение системы

CryoDAQ — система автоматического сбора данных с криогенного оборудования лаборатории АКЦ ФИАН (проект «Миллиметрон»).

**Что делает система:**
- Непрерывно считывает температуру с 24 датчиков (три монитора LakeShore 218S)
- Считывает напряжение, ток, сопротивление и мощность с источника Keithley 2604B
- Считывает давление с вакуумметра Thyracont VSP63D
- Записывает все данные в базу SQLite (один файл на каждый день)
- Следит за пороговыми значениями и подаёт тревоги
- При опасных значениях автоматически отключает нагреватель (блокировки)
- Отправляет уведомления в Telegram

**Что заменяет:**
Систему на LabVIEW, которая использовалась ранее. CryoDAQ работает надёжнее, не требует лицензии LabVIEW и может работать на любом компьютере с Windows или Linux.

**Важно:** система работает автономно. Можно закрыть и снова открыть графический интерфейс — данные не потеряются. Engine (ядро системы) продолжает работать и записывать данные.

## 2. Запуск системы

### Порядок запуска

Всегда запускайте в таком порядке:

1. **Сначала** — engine (ядро)
2. **Потом** — GUI или web-дашборд (можно оба одновременно)

### 2.1. Запуск engine (ядро системы)

Откройте терминал (командную строку) и выполните:

```
cryodaq-engine
```

На экране появятся сообщения о подключении приборов. Если всё прошло успешно, вы увидите:

```
CryoDAQ Engine запущен | приборов=5 | тревог=3 | блокировок=4
```

Каждые 30 секунд engine выводит сообщение HEARTBEAT — это означает, что система работает нормально.

**Для тестирования без приборов** (имитация):

```
cryodaq-engine --mock
```

В этом режиме система генерирует реалистичные данные без подключения к настоящим приборам.

**Чтобы остановить engine:** нажмите `Ctrl+C`. Система корректно завершит работу, сохранит все данные и отключит приборы.

### 2.2. Запуск графического интерфейса (GUI)

Откройте **второй** терминал и выполните:

```
cryodaq-gui
```

Откроется окно с вкладками. В левом нижнем углу появится индикатор подключения:
- **Зелёный кружок «Подключено»** — GUI получает данные от engine
- **Красный кружок «Отключено»** — engine не запущен или нет связи

### 2.3. Запуск web-дашборда

Откройте **третий** терминал и выполните:

```
uvicorn cryodaq.web.server:app --host 0.0.0.0 --port 8080
```

Затем откройте в браузере:

```
http://localhost:8080
```

Для доступа с другого компьютера в сети используйте IP-адрес сервера вместо `localhost`:

```
http://192.168.1.100:8080
```

Web-дашборд работает на любом устройстве с браузером, включая телефон.

## 3. Интерфейс GUI

GUI имеет пять вкладок. Переключайтесь между ними, нажимая на заголовки вверху окна.

### 3.1. Вкладка «Температуры»

Главная рабочая вкладка. Показывает:

- **Слева** — сетка из 24 карточек, по одной на каждый температурный датчик
- **Справа** — график температур за последний час

Каждая карточка показывает:
- Название канала (например, «Т7 Детектор»)
- Текущую температуру в Кельвинах
- Цветную рамку (см. раздел «Цвета» ниже)

**Нажмите на карточку**, чтобы добавить или убрать канал с графика.

### 3.2. Вкладка «Keithley»

Показывает данные источника тока Keithley 2604B:
- Напряжение (В)
- Ток (А)
- Сопротивление (Ом)
- Мощность (Вт)

### 3.3. Вкладка «Аналитика»

Показывает результаты автоматических расчётов:
- **R_thermal** — тепловое сопротивление между горячей и холодной точкой (К/Вт)
- **Cooldown ETA** — прогнозируемое время до достижения целевой температуры (секунды)

### 3.4. Вкладка «Алармы»

Таблица тревог. Каждая строка содержит:

| Столбец | Что означает |
|---------|-------------|
| Уровень | Критичность: CRITICAL (красный), WARNING (оранжевый), INFO (синий) |
| Имя | Идентификатор тревоги |
| Канал | Датчик, на котором сработала тревога |
| Значение | Текущее показание датчика |
| Порог | Пороговое значение, при превышении которого сработала тревога |
| Время | Когда сработала |
| Срабат. | Сколько раз срабатывала |
| Действие | Кнопка «Подтвердить» или текущий статус |

Тревоги отсортированы по критичности: CRITICAL — вверху, INFO — внизу.

### 3.5. Вкладка «Статус приборов»

Показывает карточку для каждого прибора:
- **Имя прибора** (LS218_1, Keithley_1, VSP63D_1)
- **Индикатор состояния** (цветной кружок)
- **Время последнего ответа**
- **Счётчик показаний и ошибок**

### 3.6. Меню

- **Файл → Экспорт CSV** — сохранить данные в текстовый файл (таблица)
- **Файл → Экспорт HDF5** — сохранить данные в научный формат HDF5
- **Эксперимент → Начать запись** — отметить начало эксперимента
- **Эксперимент → Остановить запись** — отметить конец эксперимента

### 3.7. Что означают цвета

| Цвет | Значение |
|------|----------|
| Зелёный | Всё в норме. Прибор работает, показания в допустимых пределах. |
| Жёлтый | Предупреждение. Показание выходит за норму (OVERRANGE/UNDERRANGE) или есть мелкие ошибки. |
| Красный | Ошибка. Нет связи с прибором, ошибка датчика или критическая тревога. |
| Серый | Нет данных. Прибор ещё не начал передавать данные. |

Прибор считается «без связи» (красный), если от него не было данных более **10 секунд**.

## 4. Алармы (тревоги)

### 4.1. Какие тревоги настроены

| Имя | Описание | Условие | Уровень |
|-----|----------|---------|---------|
| **overheat_sample** | Перегрев образца | Т1 Криостат верх > 100 К | WARNING |
| **cryogenics_loss** | Потеря криогеники | Т1 Криостат верх > 20 К | CRITICAL |
| **keithley_overpower** | Перегрузка нагревателя | Мощность Keithley > 5 Вт | CRITICAL |

### 4.2. Как работают тревоги

1. **Нормальное состояние (OK)** — показание ниже порога
2. **Тревога активна (ACTIVE)** — показание превысило порог. Вы видите красную/оранжевую строку в таблице алармов. Если настроен Telegram — приходит уведомление.
3. **Тревога подтверждена (ACKNOWLEDGED)** — вы нажали «Подтвердить». Тревога остаётся видна, но оператор в курсе.
4. **Тревога снята (OK)** — показание вернулось ниже порога **с учётом гистерезиса**.

**Гистерезис** — это защита от «мерцания» тревоги. Например, тревога `overheat_sample` срабатывает при 100 К, но снимается только при 95 К (порог минус гистерезис 5 К). Это предотвращает ситуацию, когда тревога то включается, то выключается при колебаниях вокруг порога.

### 4.3. Как подтвердить тревогу

1. Перейдите на вкладку «Алармы»
2. Найдите активную тревогу (строка выделена цветом)
3. Нажмите кнопку **«Подтвердить»** в правом столбце
4. Статус изменится на «Подтв.»

Подтверждение означает: «Я, оператор, вижу эту тревогу и принимаю меры». Тревога снимется автоматически, когда показание вернётся в норму.

### 4.4. Когда звонить инженеру

Звоните **немедленно**, если:

- Сработала тревога **CRITICAL** (красная)
- Тревога `cryogenics_loss` — потеря криогеники (температура криостата растёт выше 20 К)
- Тревога `keithley_overpower` — перегрузка нагревателя (мощность выше 5 Вт)
- Сработала блокировка (Keithley отключился автоматически)
- Прибор показывает статус «Нет связи» более 5 минут

Звоните в **рабочее время** (не срочно), если:

- Тревога **WARNING** (оранжевая), которая не снимается сама
- Периодические ошибки связи с приборами
- Графики показывают необычное поведение

## 5. Интерлоки (блокировки)

Интерлоки — это **автоматическая защита**. Они работают без участия оператора. Если показание датчика достигает опасного значения, система **немедленно** отключает нагреватель Keithley.

### 5.1. Какие блокировки настроены

| Имя | Условие | Действие |
|-----|---------|----------|
| **overheat_cryostat** | Любой из Т1–Т8 > 350 К | Аварийное отключение Keithley |
| **overheat_compressor** | Любой из Т9–Т12 > 320 К | Аварийное отключение Keithley |
| **detector_warmup** | Т7 Детектор > 10 К | Остановка источника тока |
| **undercool_shield** | Т5 Экран 77К < 50 К | Аварийное отключение Keithley |

### 5.2. Что происходит при срабатывании

1. Система обнаруживает опасное значение
2. **Автоматически** выполняется защитное действие:
   - `emergency_off` — полное отключение выходов Keithley (ток = 0, напряжение = 0)
   - `stop_source` — остановка программы источника тока (более мягкое отключение)
3. В лог записывается сообщение уровня **CRITICAL**
4. Если настроен Telegram — приходит уведомление

**Важно:** блокировка срабатывает один раз, после чего переходит в состояние TRIPPED. Повторное срабатывание той же блокировки возможно только после периода охлаждения (cooldown):
- overheat_cryostat / overheat_compressor: 10 секунд
- detector_warmup: 5 секунд
- undercool_shield: 30 секунд

### 5.3. Что делать после срабатывания блокировки

1. **Не паникуйте.** Нагреватель уже отключён автоматически.
2. Определите причину: посмотрите, какой датчик вызвал срабатывание (в логах engine или в Telegram-уведомлении).
3. **Позвоните инженеру** — сообщите, какая блокировка сработала и при каком значении температуры.
4. **Не перезапускайте engine**, пока инженер не разрешит — данные продолжают записываться.

## 6. Telegram-уведомления

### 6.1. Какие уведомления приходят

**Тревоги (мгновенные):**
- При активации тревоги — сообщение с эмодзи, именем тревоги, каналом, значением и порогом
- При снятии тревоги — сообщение «Тревога снята»
- Формат:

```
🔔 🚨 ТРЕВОГА

cryogenics_loss
Канал: Т1 Криостат верх
Значение: 25.3
Порог: 20
Уровень: CRITICAL
Время: 14:30:05 14.03.2026
```

**Периодические отчёты (каждые 30 минут):**
- График температур за последние 2 часа (PNG-изображение)
- Текстовая сводка: минимум/максимум/текущее значение по каждому прибору
- Список активных тревог, если есть

### 6.2. Настройка Telegram

Уведомления настраиваются в файле `config/notifications.yaml`. Для работы необходимы:
- Токен бота (получается у @BotFather в Telegram)
- ID чата или группы

Если токен не настроен (написано `YOUR_BOT_TOKEN_HERE`), уведомления не отправляются — это нормально.

## 7. Типичные сценарии

### 7.1. Начало эксперимента

1. Убедитесь, что engine запущен (в терминале видны сообщения HEARTBEAT)
2. Откройте GUI (`cryodaq-gui`)
3. Проверьте вкладку «Статус приборов» — все кружки должны быть **зелёными**
4. Проверьте вкладку «Алармы» — не должно быть активных тревог
5. В меню нажмите **Эксперимент → Начать запись**
6. Запишите время начала в лабораторный журнал

### 7.2. Штатная работа (всё зелёное)

Всё в порядке. Система работает нормально. Вы можете:
- Наблюдать за графиками на вкладке «Температуры»
- Периодически проверять вкладку «Статус приборов»
- Закрыть GUI и проверять состояние через web-дашборд на телефоне

Engine продолжает работать и записывать данные, даже если GUI закрыт.

### 7.3. Сработал аларм WARNING (оранжевый)

1. Перейдите на вкладку «Алармы»
2. Посмотрите, какой канал и какое значение
3. Нажмите **«Подтвердить»**
4. Наблюдайте за графиком — если значение возвращается к норме, тревога снимется сама
5. Если тревога не снимается в течение 10 минут — позвоните инженеру

### 7.4. Сработал аларм CRITICAL (красный)

**Действуйте немедленно:**

1. Посмотрите, какая тревога сработала
2. Если `cryogenics_loss` — температура криостата растёт. **Позвоните инженеру.**
3. Если `keithley_overpower` — мощность нагревателя превышена. Система могла автоматически отключить Keithley (через блокировку). Проверьте.
4. **Позвоните инженеру**, даже если ситуация стабилизировалась

### 7.5. Пропала связь с прибором

На вкладке «Статус приборов» карточка прибора стала **красной** с надписью «Нет связи».

Возможные причины:
- Кабель GPIB или USB отошёл
- Прибор выключен или завис
- Порт занят другой программой

Что делать:
1. Проверьте физическое подключение кабелей
2. Убедитесь, что прибор включён
3. Engine автоматически пытается переподключиться каждые несколько секунд (с нарастающей задержкой от 1 до 60 секунд)
4. Если связь не восстанавливается в течение 5 минут — позвоните инженеру

**Важно:** потеря связи с одним прибором **не влияет** на другие приборы. Остальные продолжают работать нормально.

### 7.6. Engine упал и перезапустился

Если engine завершился аварийно (сообщение об ошибке в терминале):

1. Перезапустите engine той же командой: `cryodaq-engine` (или `cryodaq-engine --mock`)
2. Engine снова подключится ко всем приборам
3. Данные, записанные до аварии, **не потеряны** (SQLite использует режим WAL)
4. GUI автоматически начнёт получать новые данные

Если engine падает повторно — позвоните инженеру и сообщите текст ошибки из терминала.

## 8. Часто задаваемые вопросы (FAQ)

**В: Можно ли закрыть GUI, не останавливая сбор данных?**
О: Да. Engine и GUI — это два отдельных процесса. Engine продолжит работать и записывать данные. GUI можно открыть и закрыть в любое время.

**В: Как посмотреть данные за прошлые дни?**
О: Данные хранятся в папке `data/` в файлах вида `data_2026-03-14.db`. Используйте экспорт в CSV или HDF5 через меню GUI: **Файл → Экспорт CSV** или **Файл → Экспорт HDF5**.

**В: Почему на графике видны «скачки» температуры?**
О: Небольшие колебания (±0.01 К) — это нормальный шум датчиков. Если скачки большие (более 1 К) — это может указывать на проблему с датчиком или контактом. Сообщите инженеру.

**В: Что означает сообщение «INTERLOCK emergency_off» в логах?**
О: Система автоматически отключила нагреватель Keithley из-за опасного значения температуры. Это штатная работа системы защиты. Позвоните инженеру.

**В: Можно ли изменить пороги тревог?**
О: Нет, это делает только инженер. Пороги задаются в файлах конфигурации `config/alarms.yaml` и `config/interlocks.yaml`. Изменение порогов блокировок без согласования — **запрещено**.

**В: Как часто система опрашивает приборы?**
О: Зависит от прибора:
- LakeShore 218S (температура): каждые 0.5 секунды
- Keithley 2604B (нагреватель): каждую секунду
- Thyracont VSP63D (вакуум): каждые 2 секунды

**В: Сколько данных хранится?**
О: Данные хранятся на диске без ограничений. Один день работы занимает примерно 50–100 МБ. При необходимости старые файлы можно переместить на внешний диск.

**В: Что делать, если web-дашборд не открывается?**
О: Проверьте, что uvicorn запущен (команда из раздела 2.3). Проверьте, что указан правильный IP-адрес. Убедитесь, что порт 8080 не заблокирован брандмауэром.

**В: Система показывает «Отключено» в GUI, но engine работает.**
О: Проверьте, что engine и GUI запущены на одном компьютере. Если на разных — нужно указать IP-адрес engine при запуске GUI (обратитесь к инженеру).

## 9. Контакты

При проблемах с системой обращайтесь:

| Вопрос | К кому |
|--------|--------|
| Система не запускается, ошибки ПО | Инженер-программист |
| Сработала блокировка, аварийная ситуация | Ответственный за криогенику |
| Аппаратные проблемы (кабели, приборы) | Инженер по оборудованию |
| Вопросы по эксперименту | Руководитель эксперимента |

**Экстренные ситуации** (сработала CRITICAL-тревога, задымление, утечка гелия):
1. Немедленно вызовите ответственного по криогенике
2. При необходимости — нажмите кнопку аварийного останова на стенде
3. Не пытайтесь самостоятельно перезапускать Keithley
