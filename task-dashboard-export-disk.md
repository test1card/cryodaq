### Строка состояния (StatusBar виджет)

Горизонтальная полоса, фиксированная высота ~40px. Содержимое:

| Элемент | Виджет | Цвет |
|---------|--------|------|
| SafetyManager state | QLabel, font bold 14pt | зелёный SAFE_OFF, жёлтый READY/RUN_PERMITTED, синий RUNNING, красный FAULT_LATCHED |
| Аптайм engine | QLabel | серый |
| Активные алармы | QLabel | серый если 0, красный если >0 |
| Keithley ON/OFF + мощность | QLabel | серый OFF, зелёный ON + P |
| Cooldown ETA | QLabel | голубой если активен, скрыт если нет |
| Свободное место на диске | QLabel | серый >10GB, жёлтый 2-10GB, красный <2GB |

Данные берутся из DerivedMetric через ZMQ: safety state, alarm count, keithley readings, cooldown_eta, disk_free.

### Карточки температур (компактные)

Каждая карточка — QFrame ~100×60px:
- **Строка 1**: имя канала (из ChannelManager), шрифт 9pt
- **Строка 2**: текущее значение, шрифт bold 14pt, «---- K» если нет данных
- **Строка 3**: тренд (▲ нагрев, ▼ охлаждение, = стабильно), dT/dt в K/ч, шрифт 8pt

Цвет фона карточки:
- Нормально: тёмно-серый
- Быстрое охлаждение (|dT/dt| > 10 K/ч): голубой оттенок
- Аларм активен: красный оттенок
- Датчик отключён (status != OK): серый с «ОТКЛ»

Сетка: 6 столбцов × 4 строки = 24 канала. Адаптивная — количество столбцов зависит от ширины окна.

### График температур

pyqtgraph PlotWidget, как в текущем temp_panel, но:
- **Переключатель масштаба времени**: кнопки [1ч] [6ч] [24ч] над графиком. 1ч = ring buffer (3600 точек), 6ч и 24ч = запрос из SQLite (decimate).
- Лог/лин переключатель для оси Y
- Видимые каналы: из ChannelManager (checkbox popup или правый клик → выбор каналов)
- **Кнопки экспорта** рядом с графиком: [📷] сохранить PNG, [📊] экспорт CSV

### Полоса давления

Узкая горизонтальная полоса (~80px высота):
- Слева: текущее значение крупно (1.2×10⁻⁵ мбар), цвет фона:
  - <1e-3 мбар: зелёный
  - 1e-3…1e-1: жёлтый
  - >1e-1: красный
- Справа: узкий pyqtgraph PlotWidget, лог-шкала Y, тот же временной масштаб что и основной график
- Синхронизация осей X с основным графиком температур

### Полоса Keithley (условная)

Видна только когда SafetyManager НЕ в SAFE_OFF. Компактная горизонтальная полоса (~50px):
- smua: V, I, R, P — четыре числа в ряд
- smub: аналогично или «OFF»
- Если SAFE_OFF → полоса скрыта (setVisible(False))

## Часть 2: Экспорт графиков и данных

### Экспорт PNG

Каждый pyqtgraph PlotWidget получает кнопку 📷 (QToolButton, иконка камеры) в правом верхнем углу:
- По нажатию: QFileDialog → save path → `plotWidget.grab().save(path, "PNG")` или `exporter = pg.exporters.ImageExporter(plotWidget.plotItem); exporter.export(path)`
- Дефолтное имя файла: `CryoDAQ_{вкладка}_{YYYY-MM-DD_HH-MM-SS}.png`
- Разрешение: 150 DPI, белый фон (для печати)

Добавить на: график температур (обзор), график давления (обзор), график Keithley, график аналитики, график теплопроводности.

### Экспорт CSV

Кнопка 📊 рядом с 📷 на каждом графике:
- По нажатию: QFileDialog → save path
- Формат: CSV, разделитель `;` (для русской Excel с запятой-десятичной), заголовки: `Время;Канал1;Канал2;...`
- Данные: то что видно на графике (ring buffer текущего масштаба)
- Десятичный разделитель: запятая (для русской локали)
- Дефолтное имя: `CryoDAQ_{вкладка}_{YYYY-MM-DD_HH-MM-SS}.csv`

### Экспорт Excel (.xlsx)

В меню Файл → Экспорт данных → Excel:
- Выбор временного диапазона (DateTimeEdit: от/до)
- Выбор каналов (чекбоксы)
- Формирует .xlsx с openpyxl:
  - Лист 1: данные (время, каналы)
  - Лист 2: информация об эксперименте (если активен)
  - Форматирование: заголовки bold, числа с 3 знаками, столбец времени в формате datetime
- Добавить `openpyxl>=3.1` в dependencies pyproject.toml

## Часть 3: Мониторинг свободного места на диске

### Engine-side: DiskMonitor

Новый файл: `src/cryodaq/core/disk_monitor.py`

```python
class DiskMonitor:
    """Периодическая проверка свободного места на диске с данными."""

    def __init__(self, data_dir: Path, broker: DataBroker, *, check_interval_s: float = 300.0):
        ...

    async def start(self) -> None:
        """Запустить периодическую проверку."""

    async def stop(self) -> None:
        """Остановить."""

    async def _check_loop(self) -> None:
        """Каждые check_interval_s: shutil.disk_usage() → DerivedMetric + Alarm."""
        # free_bytes = shutil.disk_usage(self._data_dir).free
        # Публикует DerivedMetric:
        #   channel="system/disk_free_gb", value=free_gb, unit="GB"
        # Уровни:
        #   >10 GB — OK
        #   2-10 GB — WARNING alarm через AlarmEngine
        #   <2 GB — CRITICAL alarm
```

Кроссплатформенный: `shutil.disk_usage()` работает на Windows и Linux.

### Engine wiring (engine.py)

```python
disk_monitor = DiskMonitor(data_dir=_DATA_DIR, broker=broker)
await disk_monitor.start()
# ...shutdown:
await disk_monitor.stop()
```

### Config (не нужен отдельный файл)

Пороги можно захардкодить (10 GB / 2 GB). Если нужно — добавить в `config/safety.yaml`:
```yaml
disk_monitor:
  check_interval_s: 300
  warning_threshold_gb: 10
  critical_threshold_gb: 2
```

### GUI

DiskMonitor публикует DerivedMetric → приходит в GUI через ZMQ → StatusStrip на домашней вкладке показывает «Диск: 45 GB» с цветом.

Не нужен отдельный виджет — просто QLabel в строке состояния.

### Alarms

DiskMonitor при срабатывании порога вызывает:
- WARNING: `logger.warning()` + DerivedMetric с metadata `{"alarm": "disk_low"}`
- CRITICAL: `logger.critical()` + DerivedMetric с metadata `{"alarm": "disk_critical"}`

AlarmEngine подхватит через стандартный механизм threshold alarms.
Добавить в `config/alarms.yaml`:
```yaml
- name: disk_space_warning
  channel_pattern: "system/disk_free_gb"
  condition: "< 10"
  severity: WARNING
  message: "Мало места на диске: {value:.1f} GB"

- name: disk_space_critical
  channel_pattern: "system/disk_free_gb"
  condition: "< 2"
  severity: CRITICAL
  message: "КРИТИЧЕСКИ мало места: {value:.1f} GB"
```
